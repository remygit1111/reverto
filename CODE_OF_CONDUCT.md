# Code of Conduct

Reverto is a volunteer-maintained project. To keep it
productive for everyone who participates, please follow these
guidelines.

---

## Expected Behavior

- **Be respectful.** Disagreement is fine; personal attacks are not.
- **Stay on-topic.** Issues should be about Reverto, not general
  crypto discussion or unrelated topics.
- **Be constructive.** "This is broken" is less useful than "This
  is broken, here's how I reproduced it, here's what I expected".
- **Respect maintainer time.** This is a single-maintainer
  volunteer project. Be patient with response times.
- **Use clear language.** English is the primary language for
  issues and discussions.

---

## Unacceptable Behavior

The following are not tolerated:

- **Harassment, discrimination, or personal attacks** based on any
  characteristic (race, gender, sexuality, religion, etc.).
- **Spam or self-promotion** unrelated to Reverto improvements.
- **Doxxing or sharing private information** about anyone (the
  maintainer, other contributors, or yourself).
- **Disruptive behavior** in issues or discussions intended to
  derail productive conversation.
- **Promotion of trading strategies as guaranteed returns.**
  Reverto is a tool; trading involves risk. Don't sell snake oil
  in this repository.

---

## Scope

This Code of Conduct applies to all project spaces:

- GitHub Issues
- GitHub Discussions (if enabled in the future)
- Pull Request comments (when contributions become accepted)
- Any other communication channels associated with the project

It applies whenever you are representing the project, including
when interacting with users of `app.reverto.bot` or referencing
Reverto in external venues.

---

## Enforcement

The maintainer reserves the right to:

- Edit or remove issues, comments, or other contributions that
  violate this Code of Conduct.
- Block users from interacting with the repository for serious or
  repeated violations.
- Take other action they deem appropriate.

There is no formal appeals process. This is a small project, and
the maintainer's judgment is final.

---

## Reporting

If you witness or experience behavior that violates this Code of
Conduct:

- **For most issues**: open a GitHub Issue describing the problem
  (without including private information about the violator).
- **For sensitive matters** (harassment, doxxing): use the
  GitHub Security Advisory channel described in
  [SECURITY.md](SECURITY.md). Even though it's not strictly a
  security issue, it provides a private channel.

Reports will be reviewed on a best-effort basis. The maintainer is
under no obligation to respond to bad-faith reports or attempts to
weaponize the Code of Conduct for personal disputes.

---

## Adaptations

This document adapts the spirit of the
[Contributor Covenant](https://www.contributor-covenant.org/) for
a small, single-maintainer project. It is intentionally less
